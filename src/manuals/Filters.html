Time & Variables

This section allows you to choose the filters for the chosen area of choice in the previous steps. These filters will be the criteria for your download, including: Time Period, Time Resolution, Ensemble, Emission and a list of Geographical Variables.

Time Period

Select a specific time period for your data. For example, you can select 20/05/1999 for the start date and select 10/06/2000 in the end of date. The selection means the data includes variable information between 20th May 1999 to 10th June 2000.

If you want to get just one day climate data, you can select the start date, and the end date will be automatically set as the same.

You cannot, however, choose an end date that is before the start date. You will be prompt with a warning if you try to proceed further.

Time Resulation

When selecting the time resolution, you can click the time resolution selection box with the left mouse button.

Select the desired time resolution in the pop-up drop-down menu.

The Time Resolution option consists of 3 options: Daily, Hourly and Monthly.

This function determines how many chunks will be downloaded within a timeframe chosen previously in Time Period.

E.g: A time period of 3 30-day months, with hourly time resolution will result in 3*30*24 = 2160 chunks of data.

Ensemble

The user can select an ensemble member for the result of the data. There are seven ensemble members for selection.

These ensemble members include the following:

  • CSIRO-BOM-ACCESS1-0: includes the following products (See more information at http://www.cawcr.gov.au):
    • piControl
    • amip
    • historical
    • historicalExt
    • rcp45
    • rcp85
    • abrupt4xCO2
    • 1pctCO2
  • MOHC-HadGEM2-CC: (MOHC) Met Office Hadley Centre HadGEM2-CC model output for the mid-Holocene (midHolocene) experiment. These data cover the following realms (See more information at https://catalogue.ceda.ac.uk/):
    • aerosol
    • atmos
    • land
    • landIce
    • ocean
    • ocnBgchem
    • seaIce
  • CNRM-CERFACS-CNRM-CM5: an Earth system model designed to run climate simulations. It consists of several existing models designed independently and coupled through the OASIS software developed at CERFACS (See more information at http://www.umr-cnrm.fr/).
  • CSIRO-BOM-AGCD: Australian Gridded Climate Data by Commonwealth Scientific and Industrial Research Organisation and Bureau of Meteorology (See more information at http://www.bom.gov.au//)
  • NCC-NorESM1-M: Norwegian Climate Centre (NCC) NorESM1-M model output collection. Centre for Environmental Data Analysis (See more information at https://catalogue.ceda.ac.uk/).
  • NOAA-GFDL-GFDL-ESM2M
  • MROC-MIROC5

Ensemble basically means convention and variation which different organizations or individuals of climate research use to represent their dataset. The user must be able to choose different ensemble members, which will ensure that the dataset to be downloaded will be modelled in different ways from different people, ensuring non-bias scientific results.

Emission

There are two emission options available for selection. These include: Historical – true records of climate in the past, and RCP 8.5 – forcasted records.

Geographical Variables

There are approximately 100 variables listed. The list is sorted by alphabet order, you can also search for your desired variables using the searth bar function.
